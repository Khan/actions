Move the two deep-reasoning reviewer roles from Fable 5 to Opus 5 (`claude-opus-5`): `correctness-reviewer` (which went to Fable in the 2026-07-20 A/B, buying +7 points of must-catch recall for +35% cost) and `first-principles` (which ran Fable from day one for perspective diversity). Opus 5 reports high precision *and* high recall at Opus 4.8's per-token price — half Fable's — so the recall the swap to Fable bought should be retainable without the price premium. This PR is measured by its own powered A/B; the recall/noise/verdict-agreement columns against the Fable roster are the acceptance criteria, not the release notes.

No role runs Fable now, which retires the cyber-classifier carve-out that kept the specialist lenses off Fable (Fable's safety classifiers can refuse benign security-focused analysis; a refused lens is a silent coverage hole). Nothing else in the roster moves: the orchestrator, `claim-validator`, `thread-reconciler`, `skill-auditor`, `conventions`, the opt-in whole-change reviewers, and the specialist lenses all stay on `claude-opus-4-8`, and `pattern-triage` stays on Sonnet 4.6. That is deliberate and narrower than "promote everything": the correctness reviewer's swap is the single variable the A/B measures, and moving the `claim-validator` alongside it would confound the precision columns the swap is read against. Each remaining role is its own queued arm.

**Consumers must be on gh-aw >= v0.83.0 before importing this release.** `claude-opus-5` is in no gh-aw-firewall release's curated AI-credits pricing table, nor in its bundled models.dev catalog fallback (checked through v0.27.41 and firewall `main` as of 2026-07-24), and the api-proxy's AI-credits guard — active by default — rejects any un-priced model with a 400 before the request reaches the model. That is exactly how the first-principles dispatch failed on every run of review-v1.2.0, and pinning a newer `sandbox.agent.version` cannot fix it this time because no release prices the model. The frontmatter therefore carries `models.default-ai-credits-pricing` (`input: 5.0`, `output: 25.0`), a field added in gh-aw v0.83.0: it suppresses the rejection (the guard skips it whenever a default is configured) and prices the usage at Opus 5's actual list price, so this is exact accounting rather than a fallback approximation — the proxy derives cache reads as `input x 0.1` = $0.50/M, also exact. Two caveats, both recorded in the frontmatter: the proxy's default-pricing path does not bill cache writes ($6.25/M real), so AI-credit accounting under-counts that component (the `models:` cost block carries the full rate for the run's cost summary); and the fallback applies to *any* un-priced model, so a future typo'd model id bills at Opus rates instead of failing loudly. Drop the field once a firewall release prices `claude-opus-5`. A consumer on an older gh-aw gets a compile-time failure on this file rather than a lock that 400s at runtime.

`sandbox.agent.version` stays pinned at awf v0.27.27 even though its reason for existing (Fable 5's curated pricing entry) is gone: it now only freezes sandbox behavior so this change moves exactly one variable, since dropping it would inherit the compiler's default (v0.27.37+, which flipped the no-sudo default at v0.27.32). v0.27.27 is verified to accept and map `apiProxy.defaultAiCreditsPricing`, so the fallback works under the existing pin. Removing the pin is a separate PR.

The eval harness needed one bump to be able to measure this at all: `@anthropic-ai/claude-agent-sdk` goes from `^0.3.205` to `^0.3.219`, the first release that knows `claude-opus-5`. The harness has no pricing table of its own — it enforces `--max-usd` from the `total_cost_usd` the SDK reports per result message — so an SDK that does not know the candidate arm's model turns the budget cap into a no-op on precisely the arm under test. The lockfile pinned 0.3.205 and CI installs `--frozen-lockfile`, so the range bump alone would not have taken effect. Unlike the firewall, the SDK is not in the api-proxy path, so it is only the eval that needs this; production dispatch is covered by the pricing fallback above.

The weekly drift budget is held at 220 rather than dialled back toward the pre-Fable ~$148: per-token parity with Opus 4.8 is not per-run parity (Opus 5 thinks by default and writes longer), so the case-arm-run rate is unmeasured until the first drift week on the new roster, and under-sizing costs a red run plus a contaminated noise-floor band. Re-size from a measured rate, not from the list price.
